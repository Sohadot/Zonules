## 3. Ontology — The Focus Integrity Ontology (FIO)

The governed classification of how focus is held and how it fails, across the three thesis layers.

**Top-level classes (failure-mode oriented):**

1. **Suspension failure** — the holding structures themselves fail (anatomy: zonular weakness; perception: collapse of attentional anchoring; machine: loss of calibration/grounding).
2. **Accommodation failure** — the system cannot adjust focus across conditions (anatomy: accommodation loss; perception: failure of perceptual constancy; machine: domain shift).
3. **Signal–noise failure** — the system cannot separate meaningful signal (perception: figure-ground collapse; machine: adversarial/synthetic noise).
4. **Provenance failure** — the system cannot establish whether what it sees is real (machine: deepfake/synthetic media; perception: misattributed salience).
5. **Interpretation failure** — focus is held but meaning is wrong (machine sight vs machine understanding; perceptual error).

**Entry criteria for a class:** a phenomenon enters the ontology only if (a) it is a failure or condition of *focus*, not of vision generally; (b) it maps to at least one of the three thesis layers; (c) it is source-backed or explicitly marked conceptual; (d) it connects to at least two existing reference units.

**Versioning:** FIO is versioned `MAJOR.MINOR`. Class additions are MINOR; class redefinitions are MAJOR. Every change is a `DECISION_LOG.md` entry. This is the append-aware governance that the doctrine requires.

## 4. Standard — The Focus Integrity Standard (FIS)

The ontology says what the failure modes *are*. The standard says what *intact focus integrity* looks like. A seeing system has focus integrity when:

1. Its suspension layer is identified and stable.
2. It accommodates across its intended operating range.
3. It separates signal from noise within stated tolerances.
4. It can establish provenance of what it sees to a stated confidence.
5. Held focus produces correct interpretation, not merely sharp signal.

FIS is the criterion against which the engine scores an input. It is published as a public reference page. It carries no medical-advice claims — it governs *structural reasoning about focus*, not clinical assessment of any individual's eyes.

## 5. Protocol — The Focus Integrity Assessment (FIA)

A reproducible procedure that takes a described seeing system and returns a structured reading.

- **Input:** a structured description of a system (which layer; what it is trying to focus on; reported symptoms or failure signs).
- **Steps:** (1) identify the layer; (2) test each FIS criterion against the description; (3) map each failure to an FIO class; (4) resolve each finding to the reference units that explain it.
- **Result:** a Focus Integrity reading — per-criterion status, identified failure classes, and links to the governing reference pages.

Reproducibility requirement: two operators applying FIA to the same input must produce the same class mapping. The protocol is rule-based, not interpretive.

## 6. Engine — The Focus Integrity Engine

The operational surface that runs FIA and returns a reading.

- **Phase 1 form:** fully static, client-side, deterministic. Vanilla JavaScript over a JSON encoding of FIO + FIS. **No API, no network call, no data collection, no secrets** — compliant with the No-API Doctrine and Static Security Thesis. The engine reads governed JSON and renders a result; nothing leaves the browser.
- **Input:** the user selects layer + conditions + observed signs from governed, closed vocabularies (never free-text sent anywhere).
- **Output:** a Focus Integrity reading as defined by FIA, rendered on-page.
- **Linking:** every failure class in the output links directly to its FIO class page and the relevant Layer-01/02/03 reference units. This is the doctrine's rule — outputs that point nowhere are opinions, not intelligence.
- **Medical guard:** for any anatomy-layer input the engine renders the educational-reference disclaimer and explicitly refuses diagnostic phrasing. It explains structure; it never assesses a person.
- **Deferred (Phase 3+):** a JSON endpoint / scoring API and downloadable reports — only if it can be done without violating static-first. Likely delivered as pre-generated static JSON artifacts, never a runtime service.
