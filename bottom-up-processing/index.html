Bottom-Up Processing

Bottom-up processing is a signal-reading stage

Definition

Bottom-up processing refers to perceptual analysis driven by the properties of the sensory input itself, proceeding from local features to global interpretation without prior knowledge guiding the analysis.

Scientific Grounding

Low-level features such as edges, orientations, and color contrasts are extracted in early visual cortex before categorical knowledge is applied. Bottom-up processing is captured in computational models of saliency, which predict which regions attract attention based on local feature contrast alone.

*Sources: SRC-005, SRC-006. See the source registry for classification.*

Perceptual Role

Bottom-up processing is a signal-reading stage (FIO-03): it extracts the features of the signal without prediction, providing the raw data on which top-down interpretation subsequently operates.

Relationship to the Governing Thesis

Bottom-Up Processing is one instance of the asset's governing claim: that focus is produced and held by structure that is not itself visible in the result. Its place in the focus-integrity ontology is fixed as a FIO-03 — Signal-Noise concern, the same failure class whether the seeing system is made of tissue, attention, or silicon.
