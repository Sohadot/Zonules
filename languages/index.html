English: the frozen master

The English corpus of 300 pages was frozen in Sprint 6F (2026-06-14). The freeze means:

  • All English routes, canonical URLs, and slugs are immutable
  • All claim IDs (CLM-001 through CLM-1336) and source IDs (SRC-001 through SRC-027) are frozen
  • All FIO/FIS classifications and safety classes are frozen
  • All medical disclaimers are frozen
  • Additive hreflang metadata for launched translations is permitted

The English master is the sole source of truth for every translation. No translated page may contradict a frozen English field.

Other languages: architecture defined

German, Spanish, Chinese, Arabic, Japanese, and Russian have route structures defined in data/translation-map.json and are registered in data/languages.json. No content has been translated. No pages exist. No hreflang is emitted for these languages.

Creating content for an architecture-defined language requires a governance entry in DECISION_LOG.md before any page is created.

Translation governance policy

Every translation is governed by docs/MULTILINGUAL_ARCHITECTURE.md. The core requirements:

Language-invariant fields (must not change with translation)

  • concept_id ГўВЂВ” the internal concept identifier (e.g., ZONE-ZONULES)
  • term_id ГўВЂВ” the canonical term identifier (e.g., zonules-of-zinn)
  • fio_class ГўВЂВ” the FIO failure class (e.g., FIO-01)
  • fis_criterion ГўВЂВ” the FIS standard criterion (e.g., FIS-1)
  • safety_class ГўВЂВ” the safety classification (e.g., medical-educational)
  • claim_ids ГўВЂВ” all claim traceability identifiers
  • source_ids ГўВЂВ” all source traceability identifiers

What must be preserved in every translation

  • Medical disclaimers (translated with preserved meaning ГўВЂВ” not softened, not removed)
  • The "detection is not diagnosis" boundary in AI/forensics content
  • Source attribution (SRC-001 through SRC-027 are language-invariant)
  • FIO/FIS codes (displayed as-is in all languages ГўВЂВ” they are invariant identifiers)

No partial language doctrine

A language is either launched (all governed pages complete) or it is not. There is no in-between state for production indexing. The pilot status is a governed exception allowing anchor page indexing before a full launch ГўВЂВ” it is not a workaround for the doctrine.

Validation

All translated content is validated by scripts/validate_multilingual.py and scripts/validate_all.py. These scripts enforce:

  • HTML file existence for all indexed translated routes
  • Safety marker presence in medical-educational pages
  • AI safety marker presence in deepfake/detection pages
  • FIO/FIS immutability across translations
  • No hreflang active unless language is pilot or launched and page is indexable
