Languages

Translation is not copying. Every language that carries this reference system must carry it without loss of meaning, without loss of source traceability, and without loss of safety boundaries.

English: the frozen master

The English corpus of 300 pages was frozen in Sprint 6F (2026-06-14). The freeze means:

  • All English routes, canonical URLs, and slugs are immutable
  • All claim IDs (CLM-001 through CLM-1336) and source IDs (SRC-001 through SRC-027) are frozen
  • All FIO/FIS classifications and safety classes are frozen
  • All medical disclaimers are frozen
  • Additive hreflang metadata for launched translations is permitted

The English master is the sole source of truth for every translation. No translated page may contradict a frozen English field.

French: active pilot (Sprint 7B)

French is at pilot status — a governed state between architecture-defined and fully launched. Twelve anchor pages are indexed and carry active hreflang. The pilot does not constitute a full language launch.

The twelve French anchor pages cover the three reference layers:

  • Layer 1 — Anatomy: Zonules de Zinn, Accommodation du cristallin, Rétine, Nerf optique, Cortex visuel
  • Layer 2 — Perception: Codage prédictif, Inférence active
  • Layer 3 — Machine Vision: Vision par ordinateur, Architecture Transformer, Détection des deepfakes, Provenance des images, Intégrité du Focus

A full French launch requires all 300 pages to be translated, validated, and governed by a DECISION_LOG.md entry.

Other languages: architecture defined

German, Spanish, Chinese, Arabic, Japanese, and Russian have route structures defined in data/translation-map.json and are registered in data/languages.json. No content has been translated. No pages exist. No hreflang is emitted for these languages.

Creating content for an architecture-defined language requires a governance entry in DECISION_LOG.md before any page is created.

Translation governance policy

Every translation is governed by docs/MULTILINGUAL_ARCHITECTURE.md. The core requirements:

Language-invariant fields (must not change with translation)

  • concept_id — the internal concept identifier (e.g., ZONE-ZONULES)
  • term_id — the canonical term identifier (e.g., zonules-of-zinn)
  • fio_class — the FIO failure class (e.g., FIO-01)
  • fis_criterion — the FIS standard criterion (e.g., FIS-1)
  • safety_class — the safety classification (e.g., medical-educational)
  • claim_ids — all claim traceability identifiers
  • source_ids — all source traceability identifiers

What must be preserved in every translation

  • Medical disclaimers (translated with preserved meaning — not softened, not removed)
  • The "detection is not diagnosis" boundary in AI/forensics content
  • Source attribution (SRC-001 through SRC-027 are language-invariant)
  • FIO/FIS codes (displayed as-is in all languages — they are invariant identifiers)

No partial language doctrine

A language is either launched (all governed pages complete) or it is not. There is no in-between state for production indexing. The pilot status is a governed exception allowing anchor page indexing before a full launch — it is not a workaround for the doctrine.

Validation

All translated content is validated by scripts/validate_multilingual.py and scripts/validate_all.py. These scripts enforce:

  • HTML file existence for all indexed translated routes
  • Safety marker presence in medical-educational pages
  • AI safety marker presence in deepfake/detection pages
  • FIO/FIS immutability across translations
  • No hreflang active unless language is pilot or launched and page is indexable

Next phases

Following Sprint 7B (French pilot), the planned sequence is:

  1. Sprint 7C — Expand French from 12 pilot pages toward full 300-page corpus
  2. Sprint 7D — French full launch (all 300 pages at governed status)
  3. Sprint 8 — First anchor translation in a second language (German or Spanish)

Each phase requires a governance entry in DECISION_LOG.md before content is created.
