Methodology

A reference is trusted not because it claims authority, but because its construction can be inspected. This page makes the construction inspectable.

Why this page exists

Anyone can publish text. In an environment where text is abundant and cheap, trust comes from governance that can be examined, not asserted. Zonules.com is built so that a researcher, an institution, or an AI agent can see exactly how every claim, page, and link is controlled.

One thesis, three layers

The asset is organized around a single governing thesis — focus is produced by hidden structural tension — expressed across three connected layers: the anatomy of the eye, the science of perception, and the architecture of machine vision. Every reference unit belongs to one layer and connects to the others through governed relationships, not loose association.

The Focus Integrity Ontology and Standard

The asset maintains a versioned ontology of how focus fails (the Focus Integrity Ontology) and a standard of what intact focus integrity is (the Focus Integrity Standard). These are not marketing language. They are governed structures with explicit entry criteria, version numbers, and a decision log recording every change.

Governed claims

Every scientific or technical claim on the asset is registered and classified as one of three kinds: sourced (resting on a registered reference), framework (an internal conceptual mapping, never presented as external fact), or excluded. No unsourced scientific claim is published. The References section on each reference unit shows this classification directly.

A blocking quality gate

Before any page is published, an automated quality gate checks the asset against a set of non-negotiable conditions: no broken internal links, no orphaned pages, no unregistered surfaces, no unsourced claims, no phantom or placeholder content, no unsafe medical phrasing, and no hidden infrastructure. One failed condition blocks publication. The gate is run on every change.

Static and inspectable

The public asset is static-first: no runtime API, no hidden data collection, no third-party tracking scripts, no secrets. It can be inspected, reproduced, and archived. This is part of the asset's trust posture, not a technical limitation.

For machine readers

The asset is built to be read by AI agents and citation systems as well as people: stable canonical URLs, consistent identifiers for terms, claims, and sources, structured metadata, a clean internal link graph, and an explicit distinction between sourced fact and conceptual interpretation. A claim that an agent retrieves from this asset arrives with its provenance attached.

Notes

This page describes governance, not medical or scientific content. The reference units themselves carry the relevant educational and safety framing. See the source policy for how references are classified and the glossary for the full reference index.
